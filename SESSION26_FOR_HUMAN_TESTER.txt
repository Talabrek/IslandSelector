Session 26 enhanced the existing `/islandselector admin info` command to support player lookups in addition to coordinate lookups.

### Test #87: Admin info command shows player island and slot details

**Feature:** Player information lookup for admins

**Before:** `/islandselector admin info 0,0` (coordinates only)
**After:** `/islandselector admin info Notch` (player names too!)

**What It Shows:**
- Player's grid location
- World coordinates
- Active slot number and name
- Total slots with islands (e.g., "2/5")
- List of all slots with status
- Switch cooldown (if active)

**Why This Matters:**
- Admins can quickly check any player's island status
- Essential for support tickets
- Helps diagnose slot-related issues
- Useful for auditing player data
- No need to join as the player or check database manually

**Implementation Details:**
- Enhanced: AdminInfoCommand.java (80‚Üí198 lines)
- Added: showPlayerInfo() method for player lookups
- Added: 14 new locale messages for player info output
- Smart detection: Tries coordinate first, falls back to player name
- Works for both online and offline players
- Cooldown displayed in human-readable format (e.g., "6d 23h")
