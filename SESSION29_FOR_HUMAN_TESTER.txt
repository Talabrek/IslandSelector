=================================================
HOW IT WORKS
=================================================

1. Player holds an item (e.g., diamond pickaxe)
2. Player runs: /islandselector seticon 1
3. Command gets the material name from held item
4. Saves material name to slot's iconMaterial field
5. SlotSelectionGUI displays that material as the slot icon
6. Icon persists in database (survives restarts)

Example:
- Default: Slot 1 shows GRASS_BLOCK icon
- Hold diamond pickaxe ‚Üí /islandselector seticon 1
- Now: Slot 1 shows DIAMOND_PICKAXE icon
- GUI always shows the custom icon

COMPARISON TO SETNAME (Session 28)
=================================================

Similarities:
- Same validation logic (slot number, slot exists)
- Same tab completion pattern
- Same locale message structure
- Same permission pattern

Differences:
- seticon: Takes 1 argument (slot), uses held item
- setname: Takes 2+ arguments (slot + name words)
- seticon: Checks for held item in hand
- setname: Validates name length and content

Both commands work together:
/islandselector setname 1 Mining Outpost
/islandselector seticon 1  (while holding pickaxe)
‚Üí Slot 1 shows as "Mining Outpost" with pickaxe icon
