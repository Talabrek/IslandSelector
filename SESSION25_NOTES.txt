KEY LEARNINGS - SESSION 25
=================================================

1. **Always check existing code first:**
   - Before implementing, review what exists
   - SearchListener was complete but unmarked as passing
   - Saved duplicate implementation effort
   - Lesson: Code review session is valuable

2. **feature_list.json "passes" field accuracy:**
   - "passes": false doesn't always mean unimplemented
   - Could mean: implemented but not tested
   - Need to cross-reference with source files
   - Only human tester can mark as passing

3. **Thread safety in Minecraft plugins:**
   - AsyncPlayerChatEvent runs on async thread
   - GUI operations MUST be on main thread
   - Bukkit.getScheduler().runTask() bridges gap
   - Critical for preventing concurrent modification

4. **Session management for stateful features:**
   - HashMap<UUID, Session> tracks active states
   - Must clean up after use (memory leaks)
   - Prevents multiple simultaneous operations
   - Clear on plugin disable

5. **Chat event cancellation:**
   - event.setCancelled(true) prevents broadcast
   - EventPriority.LOWEST intercepts early
   - Provides private input without public spam
   - Good UX for search/input features
