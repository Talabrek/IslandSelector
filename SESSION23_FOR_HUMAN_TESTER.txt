Session 23 implemented the grid expansion system - a critical feature for server growth:

### Tests #78-80: Dynamic Grid Expansion

**Feature:** Admin commands to expand the grid boundaries in any direction

**Commands:**
- `/islandselector admin expand east <amount>` - Add columns to the east
- `/islandselector admin expand south <amount>` - Add rows to the south
- `/islandselector admin expand west <amount>` - Add columns to the west
- `/islandselector admin expand north <amount>` - Add rows to the north

**Why This Matters:**
- Servers can start with a small grid (e.g., 10x10)
- Expand as player base grows
- No need to pre-allocate huge areas
- Prevents wasted world generation
- Critical for organic server growth

**What Changed:**
- New AdminExpandCommand with full direction support
- Grid boundaries updated dynamically
- Changes persist to config.yml immediately
- GridManager automatically uses new boundaries

TECHNICAL NOTES
=================================================

**Grid Coordinate System:**
- Centered at (0,0)
- Supports negative coordinates
- Default: -10 to 10 (21x21)
- Width = maxX - minX + 1
- Height = maxZ - minZ + 1

**Direction Mapping:**
- East = positive X (increases maxX)
- West = negative X (decreases minX)
- South = positive Z (increases maxZ)
- North = negative Z (decreases minZ)

**Config Structure:**
```yaml
grid:
  min-x: -10  # Western boundary
  max-x: 10   # Eastern boundary
  min-z: -10  # Northern boundary
  max-z: 10   # Southern boundary
```

**How It Works:**
1. Admin runs expand command
2. Settings boundaries updated
3. saveSettings() writes to config.yml
4. GridManager.isWithinBounds() uses new values
5. GUI automatically shows expanded areas
6. No server restart required (but persists across restarts)
